# Add Crop & AI Prediction Functional Specification

## Overview
After adding a land (`add_land.html`), the user lands on `add_crop.html` to decide how they want to select their crop. They can either let AI predict the best crop (`ai_crop_prediction.html`) or add it manually (`manual_crop_setup.html`).

## UI Components & Backend Requirements

### 1. Choice Page (`add_crop.html`)
- **Action:** User chooses between "AI Prediction" or "Manual Entry". Passes `land_id` in URL parameters to the next screen.

### 2. AI Crop Prediction (`ai_crop_prediction.html`)
- **Context:** The frontend displays the selected land and farm details at the top `.context-card`.
- **Top 3 Analytical Cards:**
  - **1. Potential Yield (`.yield-card`):** Shows predicted harvest amount (e.g., "120 - 135 Mon").
  - **2. Financial Forecast (`.finance-card`):** Shows estimated Expense, Revenue, and Net Profit.
  - **3. Risks & Alerts (`.risk-card`):** Shows expected weather risks (heavy rain) or pest threats based on the season.
- **Timeline Preview (`.guideline-preview`):** A summary of the major upcoming steps (Preparation, Planting, Fertilizing).
- **Backend Needed (AI Logic):**
  - **API:** `GET /api/crops/predict-analysis?land_id={id}&crop_type={crop}`
  - The backend must aggregate soil data, upcoming weather forecasts, and basic crop economics models to return JSON populating the 3 cards above.
- **Selecting the Plan:**
  - **Action:** Clicking "টাইমলাইন গ্রহণ করুন" (Accept Timeline) triggers a loading state and saves the AI-generated timeline directly to the database.
  - **Backend Needed:** `POST /api/crops/accept-ai-timeline`. Automatically creates the crop record and populates the `Tasks` table with the predicted schedule.

### 3. Manual Crop Setup (`manual_crop_setup.html`)
- **Action:** If the user opts to manually define the crop timeline rather than using AI.
- **Form Inputs (`.manual-form`):**
  - **উপকরণের ধরন (Input Type):** Radio buttons selecting 'বীজ' (Seed) or 'চারা' (Seedling).
  - **পরিমাণ (Quantity):** Number input in kg.
  - **তারিখ নির্বাচন (Date Pickers):** Uses a bottom-sheet calendar (`#calendarModal`) to pick dates for:
    - বীজ বপন বা চারা রোপণ (Planting Date)
    - প্রথম সেচ (First Irrigation)
    - প্রথম সার (First Fertilizer)
  - **Custom Steps:** Button to dynamically add more custom task dates.
  - **Health Scan Toggle (`.switch`):** Toggle to enable/disable automated disease scanning reminders every 10 days.
- **Backend needed to save Crop:**
  - `POST /api/crops/manual-setup`
  - Payload: `{ land_id, crop_type, input_type, quantity, planting_date, first_irrigation_date, first_fertilizer_date, custom_tasks: [...], enable_disease_scan: boolean }`
- **Backend Background Job:**
  - The backend receives the array of dates provided by the user and inserts them sequentially into the `Tasks` table linked to the newly created `crop_id`.

## Database Entities Used
- `Lands`
- `Crops`
- `Tasks` (Auto-generated by backend upon crop creation).
